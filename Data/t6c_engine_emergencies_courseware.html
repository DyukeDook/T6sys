3) Engine Failure in Flight

Concept: เมื่อ engine failed ต้องตัดสินใจอย่างรวดเร็วระหว่าง EjectLandAirstart โดย altitude และ energy เป็นตัวกำหนดเวลาที่มี

Indications
Power/airspeed loss, N1-Torque-ITT decay, torque ~0
Aircraft effect
Prop moves toward feather; hydraulic pressure may be insufficient
First mental model
Trade excess airspeed for altitude if appropriate
Procedure logic
Zoom/Glide 125 KIAS minimum → PCL OFF
NOTE

ถ้า engine ยังสร้าง usable torque มากกว่า 6% การ zoom อาจได้ altitude มากกว่ากรณี no-engine หลายร้อยถึงหลายพันฟุต

NOTE

ถ้าเกิด uncommanded power changes / loss of power / uncommanded propeller feather หรือ compressor stall ให้ใช้ procedure เฉพาะหัวข้อนั้น ไม่ปนกับ engine failure logic โดยตรง

CAUTION

เมื่อ hydraulic pressure หายไป gear และ flaps อาจไม่สามารถลงด้วย normal means และจะค้างที่ตำแหน่งล่าสุดขณะ engine failure; ถ้าพยายาม operate อาจเห็น unsafe/in transit

WARNING

Low altitude engine failure ต้อง prioritize aircraft control, ejection option, landing option และไม่เสียเวลาทำ troubleshooting เกินความสูง/เวลาที่มี

4) PMU Failure

Concept: PMU inoperative ไม่ได้แปลว่า engine ดับทันที แต่หมายถึง automatic management/protection ของ engine หายไปบางส่วน ต้อง monitor torque, ITT, N1 และ PCL อย่างระวัง

IndicationsPMU FAIL warning + PMU STATUS caution; อาจมี step change in engine power เมื่อ fuel management unit revert ไป manual nominal setting
Lost automationAutomatic control/limiting ของ torque, ITT, N1 หายไป
Still availablePropeller governing/overspeed protection ยังได้จาก mechanical overspeed governor
Core actionPCL minimum practical for flight → PMU OFF → reset PMU เฉพาะกรณีที่เหมาะสม → ถ้า reset ไม่สำเร็จ land as soon as practical
CAUTION

ก่อน reset PMU หรือ switch PMU OFF ให้ set power ที่ lowest practical setting เพื่อลด power shift

NOTE

ถ้า PMU failure มาพร้อม uncommanded power changes ที่ไม่ใช่ anticipated step change อย่า reset PMU; ให้ไปใช้ procedure Uncommanded Power Changes / Loss of Power / Uncommanded Propeller Feather

CAUTION

ถ้า annunciator ไม่ clear หลัง reset actions ให้ตระหนักว่า automatic torque, ITT และ N1 limiting จะไม่ available

CAUTION

เมื่อ PMU OFF จะไม่มี ground idle ระหว่าง landing rollout/taxi; ต้องวางแผน landing distance เพิ่มขึ้นเพราะ idle N1 สูงประมาณ 67%

5) Uncommanded Propeller Feather

Concept: เป็นส่วนหนึ่งของ “Uncommanded Power Changes / Loss of Power / Uncommanded Propeller Feather” ใช้เมื่อ engine ไม่สร้าง power ตามคาดหรือ prop เคลื่อนไป feather เอง ทำให้ thrust หายชัดเจน

Possible causes

PMU scheduling จาก input ผิด/จริง, oil/engine/fuel contamination, propeller dump solenoid failure, loss of oil pressure to pitch control, prop sleeve touchdown

Prop feather clues

NP ลดเร็ว, torque สูง, prop noise เพิ่ม, thrust หายชัดเจน, engine/airframe vibration; PMU FAIL/STATUS อาจขึ้นถ้า PMU drop off-line

Why PROP SYS CB?

ในกรณี errant electrical signal, การ open PROP SYS CB เอา power ออกจาก feather dump solenoid ทำให้น้ำมันกลับสู่ pitch control และ prop อาจออกจาก feather

1
PCL - MID RANGE
2
PMU SWITCH - OFF
3
PROP SYS CB - PULL ถ้า NP stable below 40%
4
PCL - As required; ถ้า power พอทำ PEL
NOTE

Mid range คือมุม PCL ทางกายภาพประมาณกึ่งกลางระหว่าง IDLE และ MAX; PCL เหนือ IDLE เพิ่มโอกาสให้ engine recover และ mid-range ช่วยลดโอกาส overtorque/overtemperature ตอน PMU OFF

CAUTION

มีโอกาส ITT limit ถูก exceed ถ้า PMU switch OFF ขณะ ITT ≥ 820 °C

CAUTION

เมื่อ PMU OFF จะไม่มี ground idle ระหว่าง landing rollout/taxi; ต้อง plan landing distance เพิ่มขึ้นจาก idle N1 ประมาณ 67%

NOTE

NP ถือว่า stable below 40% ได้เมื่อ airspeed และ torque คงที่ และไม่มี upward change นาน 3 วินาที; propeller ควรออกจาก feather ภายในประมาณ 15–20 วินาที

WARNING

ถ้า rate of descent ที่ 125 KIAS, gear/flaps/speed brake retracted และ torque 4–6% ยังมากกว่า 1500 ft/min ให้เพิ่ม torque ตามความจำเป็นถึง limit ที่ manual ระบุเพื่อให้ ROD ประมาณ 1350–1500 ft/min; ถ้า power ไม่พอให้ PCL OFF และประเมิน forced landing/ejection ตาม checklist

6) Engine Fire in Flight / Fire in Flight

Concept: FIRE annunciator หมายถึง “possibility of fire” ใน engine compartment ต้อง confirm fire ด้วยอาการประกอบก่อน shutdown engine

Fire confirmed เมื่อ FIRE annunciator + อาการประกอบเหตุผล
Smoke / flamesหลักฐานโดยตรงของ fire
Engine vibration / unusual soundsบ่งชี้ความเสียหายหรือ abnormal combustion
High ITTอาจสัมพันธ์กับ overheat/fire condition
Oil pressure, oil temperature, hydraulic pressure fluctuationบ่งชี้ความเสียหายต่อระบบใน engine compartment
If not confirmed
Initiate PEL with intention to land as soon as possible
If confirmed
PCL - OFF
Isolate
FIREWALL SHUTOFF HANDLE - PULL
Outcome
If fire extinguished: forced landing. If not / impractical: eject.
WARNING

อย่า shutdown engine จาก FIRE light อย่างเดียวโดยไม่ confirm เพราะ fire detection fault หรือ bleed air leak อาจทำให้ indication ผิดได้ แต่ถ้า confirm fire แล้วต้อง isolate engine/fuel/oil/hydraulic ตาม procedure ทันที

NOTE

ถ้า confirm fire ไม่ได้ อาจเป็น fault ของ fire warning system; ถ้าเพียง loop เดียวติดและอีก loop test good อาจเป็น false fire indication

WARNING

ในสภาพ fire/smoke ที่ aircraft control ถูกกระทบ นักบินมี option ใช้ CFS หรือ eject ตามสถานการณ์และ checklist

8) Quick Check

1. PMU Failure ต่างจาก Engine Failure อย่างไร?
PMU Failure คือ computer/automatic control ของ engine ไม่ทำงาน แต่ engine อาจยังให้กำลังได้; Engine Failure คือ engine ไม่ให้กำลัง/ flameout และต้องจัดการ energy ทันที
2. Clue สำคัญของ Uncommanded Feather คืออะไร?
NP ลดเร็ว, torque สูง, prop noise เพิ่ม, thrust หาย และอาจมี vibration; PMU FAIL/STATUS อาจขึ้นถ้า PMU drop off-line
3. ทำไม PMU OFF ต้องระวัง landing distance?
เพราะ ground idle ไม่ available และ idle N1 สูงประมาณ 67% ทำให้ rollout/taxi/landing distance เพิ่มขึ้น
4. FIRE annunciator ติดอย่างเดียวพอจะ shutdown engine ทันทีหรือไม่?
ตาม concept ของ manual ต้อง confirm fire ด้วยอาการประกอบก่อน เพราะอาจเป็น false indication หรือ fire detection fault
5. ทำไม Engine Failure ต้องคิดเรื่อง zoom/glide?
เพราะ excess airspeed สามารถแลกเป็น altitude เพื่อเพิ่ม terrain clearance, glide range, เวลาทำ airstart หรือ ejection margin
