๑.๑ Uncommanded Prop Feather

Loss of thrustNP decreaseHigh torquePROP SYS CB

Uncommanded Prop Feather คือกรณีที่ใบพัดเข้าสู่ทิศทาง feather โดยไม่ได้สั่ง ทำให้ thrust หายไปอย่างชัดเจน ทั้งที่ engine บางส่วนอาจยังทำงานอยู่ สาเหตุหนึ่งที่คู่มืออธิบายคือ feather dump solenoid ได้รับสัญญาณไฟฟ้าผิดพลาด ทำให้น้ำมันไปควบคุมกลไก pitch ของใบพัดลดลง แล้วใบพัดไหลไปทาง feather

Mental model: Prop feather โดยไม่ตั้งใจ = thrust หายเพราะ prop ไม่ bite อากาศแล้ว เป้าหมายคือเอา prop ออกจาก feather และคง controllability ให้ได้

Indications ที่เด่น

  • NP ลดลงอย่างรวดเร็ว
  • Torque สูงผิดสัดส่วน
  • Propeller noise เพิ่มขึ้น
  • สูญเสีย thrust อย่างเห็นได้ชัด
  • อาจมี vibration ของเครื่อง/airframe

ถ้า PMU หลุด offline

  • PMU FAIL warning และ PMU STATUS caution จะติด
  • อาจเกิด step change ของ power เมื่อ fuel metering unit revert ไป manual nominal condition
  • ยังต้องคิดเรื่อง landing distance เพิ่ม เพราะ ground idle ไม่พร้อมเมื่อ PMU OFF

Core checklist logic

  1. PCL - MID RANGE ตำแหน่งประมาณกึ่งกลางระหว่าง IDLE กับ MAX
  2. PMU switch - OFF
  3. PROP SYS circuit breaker - PULL, if NP stable below 40%
  4. PCL - As required
ขั้นตอนทำไปทำไมข้อควรระวัง
PCL ที่ mid rangeช่วยให้เครื่องมีโอกาส recover ดีขึ้น และลดความเสี่ยง overtorque/overtemp ตอน PMU OFFต่ำเกินไปอาจไม่มีโอกาส recover, สูงเกินไปเสี่ยงเกิน limits
PMU OFFเพิ่ม oil pressure/volume ไป prop pitch control ใน manual mode อาจพอเอาชนะ localized oil pressure loss ได้ถ้า ITT ≥ 820°C มีโอกาส exceed ITT limits เมื่อ switch OFF
Pull PROP SYS CBตัดไฟไป feather dump solenoid ถ้าสาเหตุเกิดจากสัญญาณไฟฟ้าผิดพลาดทำเมื่อ NP stable below 40% ตามเกณฑ์
คู่มือระบุ: เมื่อ pull PROP SYS circuit breaker แล้ว ใบพัดควรเริ่มออกจาก feather ภายในประมาณ 15-20 วินาที
ข้อจำกัดภาคพื้น: เมื่อ PMU OFF, ground idle จะไม่ available ระหว่าง landing rollout / taxi จึงต้องวางแผน landing distance เพิ่มจาก idle N1 ที่สูงกว่าเดิม
จำง่าย: “Mid range — PMU OFF — NP ต่ำคงที่ค่อย pull PROP SYS — แล้วคุม PCL ต่อ”

๑.๒ Uncommanded Power Change

RollbackLoss of expected powerPMU schedulingPossible engine/prop/fuel issue

หัวข้อนี้ใช้เมื่อเครื่องมี power change ที่ไม่ได้สั่ง หรือ engine ไม่ให้กำลังตามที่ควรได้ ปัญหาอาจมาจาก PMU scheduling, faulty sensor inputs, oil/engine/fuel contamination, propeller dump solenoid failure หรือ propsleeve touchdown ซึ่งบางกรณีก็ลามไปเป็น uncommanded prop feather ได้

Mental model: อย่าด่วนสรุปว่าเป็น engine flameout อย่างเดียว — ให้คิดเป็นกลุ่ม “power available ไม่ตรงกับที่ควรเป็น” แล้วใช้ procedure เดียวกันเพื่อลดความเสียหายและเอา power กลับมา
1. Recognizeสังเกต thrust ลด, fuel flow ต่ำกว่าคาด, N1 ลดโดยไม่สั่ง หรือมี surges / uncontrollable high power
2. Stabilizeคุม airspeed, aircraft attitude และพิจารณา nearest suitable field ทันที
3. Use the checklistprocedure นี้ครอบคลุม uncommanded power changes, loss of power และ uncommanded prop feather
4. Think PELถ้า power ไม่น่าเชื่อถือ ให้คิดแบบ impending engine failure
5. Monitor trendดู NP, torque, ITT, N1, fuel flow และ vibrations ตลอดเวลา
6. Decide earlyถ้า power ไม่พอให้คง descent rate ที่ต้องการ ให้เตรียม PCL OFF / forced landing / eject ตามสถานการณ์
สิ่งที่ช่วยแยกโรค: ถ้ามีเสียงดังผิดปกติ, backfire, flames หรือ smoke อาจไปทาง compressor stall มากกว่า ส่วน uncommanded power change มักเด่นที่ power/thrust ไม่สัมพันธ์กับการสั่งและ engine indications
จากคู่มือ: ถ้า stabilized at 125 KIAS, gear/flaps/speed brake retracted และ 4-6% torque แล้ว VSI ยังมากกว่า 1500 ft/min ให้เพิ่ม torque ตามจำเป็น (สูงสุดถึง 131%) เพื่อพยายามคงประมาณ 1350-1500 ft/min; ถ้ายังไม่พอให้ set PCL OFF
จุดอันตราย: ถ้า PMU problem ทำให้ power change แปลก ๆ อย่าพยายาม reset แบบสุ่มโดยไม่ลด power ก่อน เพราะอาจทำให้เกิด step change ที่รุนแรงกว่าเดิม
จำง่าย: “Power ไม่มาตามสั่ง = บินให้ได้ก่อน, คิด PEL, monitor trend, ตัดสินใจลงเร็ว”

๑.๓ Trim System Malfunction

Trim runaway / stuckTrim interruptTRIM DISCONNECTTAD

Trim system malfunction คือกรณีที่ trim ไม่ทำงาน, ค้าง, หรือทำงานผิดปกติ ทำให้แรงที่นักบินต้องถือควบคุมเพิ่มขึ้น แม้จะไม่ใช่ emergency ใหญ่แบบ engine failure แต่ถ้าปล่อยไว้จะเพิ่ม workload มาก โดยเฉพาะช่วง approach/landing

Mental model: Trim เสีย = เครื่องยังบินได้ แต่ pilot ต้อง “เอาอำนาจกลับมาอยู่ในมือ” โดยตัด trim ที่มีปัญหาออก แล้วบินด้วยแรงถือแทน

สิ่งที่ควรรู้เรื่องระบบ

  • กด trim interrupt button หรือ set trim disconnect switch ไป disconnect จะตัด trim systems ทั้งหมดและตัด TAD ด้วย
  • ถ้าดึง circuit breaker ของ trim ใด trim นั้นจะใช้ไม่ได้
  • ถ้าดึง rudder trim CB จะเสีย automatic TAD correction inputs ไปด้วย

Stuck/Failed Trim checklist

  1. TRIM DISCONNECT switch - NORM
  2. AIL/EL TRIM หรือ RUD TRIM circuit breaker(s) - Check, reset if open
ชนิดปัญหาผลที่เกิดแนวคิดแก้
Elevator trim failstick force เพิ่ม โดยเฉพาะหลังลดความเร็วลงถือแรงด้วยมือและวางแผน approach/landing อย่างรู้ตัว
Aileron trim failแรง lateral ถือเพิ่มขึ้นบินด้วยมือ, trim ไม่ช่วย แต่ยังควบคุมได้
Rudder trim / TAD failต้องใช้ rudder เองมากขึ้นเมื่อ power/configuration เปลี่ยนเพิ่ม pilot workload แต่ยังไม่ทำให้ mission impossible ทันที
ตัวเลขที่คู่มือบอก: ถ้า elevator trim fail ระหว่าง high-speed cruise/descent พอลดความเร็วเพื่อ approach/landing stick force จะขึ้นประมาณ 10 ปอนด์; aileron trim ประมาณ 5 ปอนด์; rudder trim ประมาณ 20 ปอนด์
ถ้าเป็น TAD FAIL: หมายถึง automatic rudder trim correction หายไป ภาระนักบินเพิ่มขึ้น แต่ไม่มีผลอื่นต่อ mission capability มากนัก และอาจลอง reset TAD circuit breaker ตาม procedure ได้
จำง่าย: “Trim เสีย = ตัดระบบที่เพี้ยน, ถือเครื่องด้วยมือ, บริหาร workload ให้ดี”

๑.๔ Suspected Fuel Leak

Leaking Fuel from WingFuel balanceTo non-leaking tankLand ASAP

หัวข้อนี้ใน manual ใกล้เคียงกับ Leaking Fuel from Wing ใช้เมื่อต้องสงสัยว่ามีน้ำมันรั่วจากปีก ไม่ว่าจะจาก birdstrike หรือ system failure เป้าหมายคือ ลดการสูญเสียน้ำมันรวม และกันไม่ให้ระบบยัง feed น้ำมันไปยังฝั่งที่รั่วต่อไป

Mental model: Fuel leak = ต้องหาว่ารั่วจริงไหม, รั่วฝั่งไหน, แล้วสั่งให้ระบบไปใช้ tank ที่ไม่รั่วเพื่อลดการสูญเสีย total usable fuel
1. Suspect itสงสัยได้จากเห็นการรั่วด้วยตา, fuel quantity ลดผิดปกติ, imbalance เพิ่ม หรือ FUEL BAL caution ร่วมกับลักษณะการใช้ fuel แปลก ๆ
2. Inspectvisual inspect aircraft structure ถ้าสถานการณ์อำนวย ว่ามี sign ของ leakage หรือไม่
3. Reconfigure fuel balanceเข้าระบบ manual fuel balance เพื่อให้ feed ไปฝั่ง non-leaking tank
4. Monitorดู fuel gauges, trend การลดลงของ fuel และ controllability จาก imbalance
5. Stop transfer when leaking tank emptyเมื่อฝั่งรั่ว empty แล้วให้ OFF manual balance
6. Land as soon as possibleเพราะ usable fuel กำลังหายอย่างควบคุมไม่ได้

Core checklist logic (Leaking Fuel from Wing)

  1. Aircraft structure - Visually inspect for signs of leakage
  2. If leaking fuel overboard: FUEL BAL switch - MAN/RESET
  3. MANUAL FUEL BAL switch - To non-leaking tank
  4. When leaking tank is empty: MANUAL FUEL BAL switch - OFF
  5. Land as soon as possible
ประเด็นสิ่งที่ต้องเข้าใจ
ทำไมต้องไปที่ non-leaking tankเพื่อหยุดส่ง fuel เพิ่มไปยังฝั่งที่กำลังรั่ว และเก็บ usable fuel ที่เหลือไว้ให้มากที่สุด
ทำไมยังต้อง monitor balanceเพราะ full lateral imbalance ยังบินได้ แต่ stick force จะเพิ่มขึ้น โดยเฉพาะถ้า crosswind
ทำไมต้องลงเร็วเพราะ fuel quantity และ fuel planning ปกติจะเชื่อถือได้น้อยลงเมื่อมี leak จริง
เกี่ยวข้องกับ Fuel Imbalance: ก่อนจะพยายาม balance fuel ด้วยตนเอง ควร verify imbalance และ check for fuel leaks ก่อนเสมอ เพราะถ้ารั่วจริง การ balance ผิดทิศจะยิ่งทำให้สูญเสียน้ำมันเร็วขึ้น
อย่าสับสนกับ FP FAIL: ถ้า FP FAIL caution ติด ไม่ควรพยายาม manual balance แบบเดาสุ่ม เพราะ probe failure อาจทำให้ข้อความ imbalance ไม่ถูกต้อง และ manual balancing อาจทำให้ปัญหาแย่ลง
จำง่าย: “Fuel leak = หาให้เจอว่ารั่วข้างไหน แล้ว feed ไปข้างที่ไม่รั่ว จากนั้นรีบลง”
