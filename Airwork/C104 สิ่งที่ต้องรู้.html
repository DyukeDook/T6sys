๑.๑ Fuel System Failure and Fuel Balancing Procedure

Fuel feedBoost pumpFuel imbalanceLand as soon as practical

หัวข้อนี้ต้องแยกเป็น 2 เรื่อง: fuel system failure และ fuel balancing เพราะอย่างแรกคือความผิดปกติของระบบจ่ายเชื้อเพลิง ส่วนอย่างหลังคือการจัดการเชื้อเพลิงซ้าย-ขวาให้สมดุลเพื่อลด asymmetry และคงการ feed ให้เหมาะสม

Mental model: Fuel problem = ตรวจว่ามี fuel จริงไหม, feed ปกติไหม, imbalance มากไหม, และมันกำลังพาไปสู่ engine malfunction หรือไม่

สิ่งที่ต้องนึกถึงเมื่อเกิด fuel malfunction

  • Fuel quantity ที่เห็นเชื่อถือได้หรือไม่
  • Fuel flow สูงผิดปกติ / fluctuation หรือไม่
  • มี warning/caution เช่น FP FAIL, low fuel หรือ indication related fuel pump หรือไม่
  • เครื่องยนต์เริ่มมีอาการ power loss, surging หรือ flameout tendency หรือไม่

เป้าหมายของ fuel balancing

  • ลด lateral imbalance
  • ทำให้ feed จาก tank ที่มี fuel มากกว่าถูกใช้ก่อน
  • คง controllability และลด trim burden
  • ป้องกันไม่ให้ imbalance ไปไกลจนจำกัดการบิน/landing
อาการ/ประเด็นความหมายทางระบบผลกับการตัดสินใจ
Fuel flow สูงมากผิดปกติอาจบ่งชี้ fuel system malfunction และมีความเป็นไปได้ของ engine failureเตรียม nearest suitable field และมองภาพแบบ engine problem ไม่ใช่แค่ quantity issue
Fuel quantity ซ้าย-ขวาไม่เท่ากันเกิด imbalance / automatic balancing อาจไม่ทำงานเต็มที่ต้องวางแผน balance และ monitor controllability
Fuel pressure/pump faultการ feed เชื้อเพลิงอาจไม่เป็นไปตามปกติตาม checklist, ลดความเสี่ยง และเตรียมลงสนาม

Study logic

  1. Confirm indications ดู quantity, fuel flow, engine response, warning/caution
  2. If engine-related symptoms exist, think PEL / nearest suitable field
  3. Minimize unnecessary configuration changes
  4. If imbalance exists, use the appropriate balancing procedure / tank selection logic per checklist
  5. Recheck trend ว่า imbalance ดีขึ้นหรือแย่ลง
  6. Plan recovery / landing before the problem becomes an engine problem
ข้อคิดสำคัญ: ถ้า fuel issue มาพร้อมอาการ engine roughness, power loss, surging หรือ abnormal fuel flow ให้เปลี่ยน mindset ไปทาง “impending engine failure” ทันที
จำง่าย: “Fuel มีไหม — Feed ดีไหม — Balance หรือยัง — ถ้าเริ่มกระทบ engine ให้รีบลง”

๑.๒ Hydraulic System Failure

GearFlapBrake/NWS effectEmergency extension

ระบบไฮดรอลิกใน T-6C มีผลกับอุปกรณ์หลายอย่างที่เกี่ยวข้องกับการลงจอดและการควบคุมภาคพื้น เช่น landing gear actuation, flap operation, nosewheel steering และ braking characteristics บางส่วน ดังนั้นเมื่อ hydraulic system fail ประเด็นใหญ่ไม่ใช่ “บินต่อได้ไหม” อย่างเดียว แต่คือ “จะลงอย่างไร”

Mental model: Hydraulic failure = คิดถึง landing configuration ทันที: gear จะลงปกติไหม, flap ใช้ได้ไหม, braking/NWS จะเป็นอย่างไร
1. Aviateรักษา aircraft control และประเมินว่ามีผลต่อ controllability ระหว่างบินหรือไม่
2. Diagnoseดู indications, affected systems และ secondary effects เช่น gear/flap indications
3. Configure late and deliberatelyอย่ารีบเปลี่ยน configuration ถ้ายังไม่พร้อม เพราะอาจเสีย option
4. Plan the landingเลือกสนามยาวพอ, ลมเหมาะ, emergency services พร้อม
5. Brief the techniqueno flap / emergency gear / directional control / brake use
6. Prepare for follow-on problemsgear unsafe, high landing speed, long roll, steering issues
ระบบย่อยที่อาจได้รับผลผลที่นักบินจะรู้สึกสิ่งที่ต้องคิดต่อ
Landing geargear อาจไม่ลงปกติ หรือ indication ไม่สมบูรณ์เตรียมใช้ emergency extension / gear unsafe landing plan
Flapsflap อาจค้างหรือใช้ไม่ได้landing speed สูงขึ้น, touchdown distance ยาวขึ้น
Steering / brakingการคุมทิศทางบน runway อาจยากขึ้นเลือก runway กว้าง/ยาว และใช้ brakes อย่างระมัดระวัง
หลุมพรางที่พบบ่อย: รอวางแผน landing ช้าเกินไป ทั้งที่ hydraulic malfunction ส่วนใหญ่ “บทหนักสุด” จะมาในช่วง approach/landing มากกว่าระหว่าง cruise
สนามที่เหมาะ: runway ยาว, crosswind ต่ำ, surface ดี, rescue พร้อม และมีเวลา setup straight-in approach
จำง่าย: “Hydraulic fail = คิดถึง gear, flap, brake, steering แล้วเลือกสนามให้ดี”

๑.๓ OBOGS Failure

OxygenPhysiological symptomsEmergency oxygenDescend

OBOGS คือ On-Board Oxygen Generating System ทำหน้าที่สร้างออกซิเจนจาก conditioned bleed air ให้กับนักบินทั้งสอง ถ้า OBOGS fail นักบินยังอาจหายใจอากาศใน cockpit ได้ผ่าน anti-suffocation valve แต่ความเข้มข้นออกซิเจนจะไม่เพียงพอเมื่ออยู่ที่ความสูงสูง ๆ ดังนั้นหัวข้อนี้ต้องคิดเป็นทั้ง system failure และ physiological emergency

Mental model: OBOGS failure = เชื่อร่างกายมากกว่าแค่ไฟเตือน ถ้ามีอาการ hypoxia ให้แก้เหมือน physiological emergency ทันที

Indications

  • OBOGS FAIL warning / caution indications
  • Flow indicator/blinker ผิดปกติ
  • หายใจลำบาก, เวียนหัว, มึนงง, tunnel vision
  • white dust / respiratory irritation ซึ่งอาจบ่งชี้ concentrator malfunction

Action mindset

  • Secure mask and regulator connection ก่อน
  • ใช้ emergency oxygen source ถ้าจำเป็น
  • descend to safe altitude
  • terminate mission / land as required

Study logic

  1. Check mask fit and oxygen connections บางครั้ง fault เกิดจาก mask down/loose
  2. If symptoms persist, treat as real oxygen problem
  3. Select / activate emergency oxygen as required
  4. Descend ไป altitude ที่ลดความต้องการออกซิเจน
  5. Communicate and terminate the sortie
ข้อสำคัญ: inflight engine failure หรือ total electrical power loss อาจทำให้ bleed air supply ไป OBOGS หาย และกลายเป็น OBOGS failure ตามมาได้
อย่าหลงเชื่อแค่อาการชั่วคราว: hypoxia มักทำให้ judgment แย่ลง ดังนั้นถ้ารู้สึก “แปลก” ให้ลงมือแก้ก่อนคิดนาน
จำง่าย: “Mask — Emergency oxygen — Descend — Land”

๑.๔ Electrical Failure

Battery/GeneratorLoad sheddingEssential systemsAvionics impact

Electrical failure คือหัวข้อใหญ่เพราะมันไม่ใช่แค่ไฟหาย แต่หมายถึง หลายระบบจะล้มตาม bus ที่ดับ เช่น displays, radios, avionics, lights, sensors หรือ systems support อื่น ๆ ความเข้าใจหลักคือแยกก่อนว่าเป็น generator problem, battery problem หรือ total electrical failure

Mental model: Electrical problem = รู้ว่า bus ไหนยังอยู่, ของสำคัญอะไรยังมี, battery จะอยู่ได้นานแค่ไหน และต้องลงเมื่อไร
ชนิดปัญหาผลที่เป็นไปได้แนวคิด
Generator / charging problemระบบยังทำงานจาก battery ได้ชั่วคราว แต่ endurance จำกัดลด electrical load และรีบไปสนาม
Battery / bus issueบาง display/annunciations อาจหายหรือ resetmonitor essential indications และเตรียม partial panel/standby use
Total electrical failureสูญเสีย avionics/display หลายรายการ, radios อาจใช้ไม่ได้ใช้ standby references, วางแผนลงเร็ว และคาด OBOGS impact
1. Confirmดู volt/amp, cautions, GEN/BAT indications และสิ่งที่หลุดไป
2. Shed loadตัดอุปกรณ์ที่ไม่จำเป็นเพื่อยืด battery life
3. Preserve essentialsเหลือเฉพาะสิ่งที่จำเป็นต่อ navigate / communicate / land
4. Plan for degraded avionicsคาดว่า MFD, HUD, radios หรือ nav aids บางส่วนอาจหายตาม bus
5. Consider follow-on failuresโดยเฉพาะ OBOGS/bleed-support implications ในบางสถานการณ์
6. Land before battery is goneอย่ารอให้กลายเป็น total failure โดยไม่ตั้งใจ
Priority checklist ในหัว: flight path → standby references → one good comm if available → nearest suitable field
หลุมพราง: พยายาม keep everything on ไว้หมดเพราะกลัวเสีย situational awareness สุดท้าย battery หมดเร็วกว่าเดิม
จำง่าย: “รู้ว่าอะไรยังมี — ลดโหลด — เก็บไฟไว้ให้ของจำเป็น — ลงก่อนมืดหมดทั้งลำ”

๑.๕ Avionic Failure

IACDisplay reversionLoss of HUD/MFD functionMission impact

Avionic failure แตกต่างจาก electrical failure ตรงที่ไฟอาจยังมี แต่ computer/display chain บางส่วน fail เช่น IAC, MFD, ADC inputs, navigation integration หรือ display processor ส่งผลให้ข้อมูลบางอย่างหาย, ซ้ำซ้อน หรือถูก reversion ไปแสดงที่อื่น

Mental model: Avionics fail = ไม่ได้แปลว่าบินไม่ได้ทันที แต่ต้องรู้ว่าจะดูข้อมูลอะไรจากที่ไหนแทน

สิ่งที่มักเกิดเมื่อ avionics fail

  • HUD หายหรือ repeater mode เปลี่ยน
  • MFD หนึ่งจอหรือหลายจอ blank / freeze / degraded
  • NAV/Map/Sensor integration ผิดปกติ
  • บาง reversion mode แสดงข้อมูลจำเป็นแบบจำกัด

สิ่งที่นักบินต้องทำ

  • cross-check standby instruments
  • reconfigure display ให้เหลือข้อมูล primary flight / engine / nav ที่จำเป็น
  • ลด task loading
  • brief crew ว่าใคร monitor อะไรจากจอไหน
Failure typeผลต่อการใช้งานแนวคิด
Single display failureข้อมูลบางหน้าเรียกไม่ได้ที่จอนั้นย้าย page ไปจออื่น และลด nonessential pages
IAC / processing failureHUD, repeat mode, integrated navigation หรือ symbol generation อาจหายหลายส่วนบินด้วย remaining displays/standby และ terminate mission
Sensor/data source failureข้อมูล air data / nav / heading อาจไม่ตรงidentify bad source แล้วใช้ source ที่เชื่อถือได้กว่า
CRM สำคัญมาก: ในเครื่องสองที่นั่งต้องแบ่งหน้าที่ให้ชัด เช่น front คุม aircraft, rear จัด display/COM/navigation troubleshooting
จำง่าย: “Avionics fail = หา source ที่ยังเชื่อถือได้ แล้วบินจากตรงนั้น”

๑.๖ HUD Symbology

Flight pathAirspeed/AltitudePitch ladderScan discipline

หัวข้อนี้ไม่ใช่ emergency โดยตรง แต่เป็น “พื้นฐานที่ต้องรู้” เพราะถ้าเข้าใจ HUD symbology ดี นักบินจะบริหาร energy, attitude และ flight path ได้แม่นขึ้น โดยเฉพาะในช่วง takeoff, rejoin, aerobatics entry, instrument references และ landing

Mental model: HUD ไม่ได้แทนสมองนักบิน — มันแค่เอาข้อมูลสำคัญมาไว้ใน line of sight เพื่อให้ scan สั้นลง
Waterline / reference lineใช้เป็น reference attitude ของเครื่อง ไม่ใช่ flight path จริงเสมอไป
Flight Path Marker (FPM)บอกว่า aircraft “กำลังไปทางไหน” บนโลกจริง ถ้า crosswind/side slip จะไม่ตรงกับหัวเครื่อง
Pitch ladderช่วยประเมิน pitch attitude และ bank relation
Airspeed tape / cueช่วยคุม energy โดยไม่ต้องก้มลงดูตลอด
Altitude / VSI / heading dataเป็น core information สำหรับ scan แบบ outside-inside
Steering / mode-dependent symbologyบาง symbol จะขึ้นตาม nav/approach/training mode ที่เลือกอยู่
Symbol / กลุ่มข้อมูลใช้ดูอะไรข้อคิดเวลาใช้งาน
FPMflight path จริง, approach aim, climb pathถ้าต้องการให้เครื่องไปที่ไหน พยายามวาง FPM ไปที่นั่น
Pitch ladderattitude awarenessอย่าใช้แทน outside references ทั้งหมด
Airspeed / Altitudeenergy control และ trendต้องอ่าน trend ร่วมกับ attitude ไม่ใช่ดูตัวเลขอย่างเดียว
Heading / steering cuesnavigation / interceptรู้ว่า cue นั้นมาจาก mode อะไร เพื่อไม่ตีความผิด
เทคนิคการจำ HUD: แยกเป็น 3 กลุ่ม — “Where the nose is” (attitude), “Where the jet is going” (FPM), และ “How the energy is changing” (airspeed/altitude/trend)
ข้อผิดพลาดที่พบบ่อย: fixation กับ HUD จนไม่ scan outside/standby หรืออ่าน FPM กับ waterline สลับความหมายกัน
จำง่าย: “HUD = Attitude + Flight Path + Energy”

สรุปจำเร็วทั้ง C104

Fuelดู quantity, feed, flow และ imbalance — ถ้ากระทบ engine ให้คิด nearest suitable field
Hydraulicอย่ามองแค่ตอนบิน ให้คิด landing gear / flap / steering / brake ตั้งแต่ต้น
OBOGSเช็ค mask, ใช้ emergency oxygen ถ้าจำเป็น, descend และ land
Electricalรู้ว่าอะไรยังมี, shed load, เก็บไฟให้ของจำเป็น และลงก่อน battery หมด
Avionicsหา source/display ที่ยังเชื่อถือได้ แล้วจัด scan ใหม่
HUDเข้าใจ FPM, attitude และ energy เพื่อใช้เป็นเครื่องมือ ไม่ใช่เป็นสิ่งรบกวน
