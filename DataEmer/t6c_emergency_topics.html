๑. Emergency Ground Egress

GroundEscapeEjection Seat Safety

ความหมาย: การออกจากเครื่องแบบฉุกเฉินขณะอยู่บนพื้น หลังเครื่องหยุดหรือจำเป็นต้องรีบออกจาก cockpit เช่น ไฟไหม้, crash landing, prop strike, aircraft damage หรือ canopy เปิดไม่ได้

Logic: ทำให้ seat ปลอดภัย → เปิดทางออก → ปลดตัวเอง → ตัดไฟ → ออกจากเครื่อง

ลำดับจำง่าย

  1. ISS mode selector - SOLO ป้องกัน seat อีกตำแหน่งถูก command โดยไม่ตั้งใจ
  2. Seat safety pin - Install ใส่ pin ให้ seat ที่มีคนอยู่
  3. Parking brake - As required ป้องกันเครื่องเคลื่อน ถ้าสถานการณ์ทำได้
  4. Canopy - Open ใช้ทางออกปกติ
  5. CFS - Use if required ถ้า canopy เปิดไม่ได้หรือจำเป็นต้องออกทางขวา
  6. Release straps / belt / fittings / leg restraints ปลดทุกอย่างที่ยึดตัว
  7. BAT / GEN / AUX BAT - OFF ตัด electrical power
  8. Evacuate aircraft ออกจากเครื่อง
ระวัง: ถ้าใช้ Canopy Fracturing System ควรเข้าใจว่ามีเศษ canopy และ fragments กระเด็น ต้องป้องกันใบหน้า/ตา และทำตาม procedure ของหน่วย
จำเร็ว: SOLO → PIN → BRAKE → CANOPY/CFS → STRAPS → POWER OFF → OUT

๒. Fire Warning on Ground
(Fire annunciator illuminated)

FireGround OperationShutdown

ความหมาย: มีหลักฐานว่าเกิด engine fire ขณะอยู่บนพื้น เช่น ตอน start, taxi, run-up หรือ ground operation

สิ่งที่เห็นได้

  • FIRE annunciator illuminated เป็น primary indication
  • เห็น smoke หรือ fire
  • Engine indication ผิดปกติ เช่น ITT สูง, fuel flow ผิดปกติ
  • Ground crew / tower / aircraft อื่นแจ้งว่ามีไฟ
Logic: Fire on ground ไม่ใช่ troubleshooting ยาว เป้าหมายคือ shutdown → isolate → egress ถ้าจำเป็น

ระบบที่เกี่ยวข้อง

ระบบเกี่ยวข้องอย่างไร
Fire Detectionตรวจจับความร้อน/ไฟและแจ้งเตือน FIRE annunciator
Fuel Systemfuel เป็นแหล่งพลังงานของไฟ ต้องตัดเมื่อมี fire
Firewall Shutoffใช้ isolate fuel, hydraulic fluid และ bleed air บริเวณ engine/firewall
จำเร็ว: Fire on ground = Shutdown + Isolate + Egress if required

๓. Abort Start

StartHot StartHung StartNo Start

ความหมาย: การยกเลิก engine start เมื่อ start กำลังผิดปกติ เช่น hot start, hung start, no start หรือมีปัญหา battery/PMU/start sequence

เมื่อไหร่ควร Abort Start

  • ITT เพิ่มเร็วและมีแนวโน้มเกิน limit
  • ITT สูงผิดปกติค้างอยู่นานเกินเกณฑ์
  • N1 เพิ่มแล้วหยุดหรือไม่เพิ่มต่อ เป็นลักษณะ hung start
  • มี fuel flow แล้ว แต่ ITT ไม่ rise ในเวลาที่ควร เป็นลักษณะ no start
  • เกิด red BATT BUS warning ระหว่าง start
  • PCL ถูกขยับหรือ start sequence condition เปลี่ยนผิดจังหวะ
Logic: ถ้า start ไม่ปลอดภัย ให้หยุดการจ่าย fuel / หยุด sequence แล้วทำ motoring run ถ้ามี fuel เข้าเครื่อง

Motoring Run คืออะไร

คือการหมุน engine โดยไม่ให้เกิด combustion เพื่อ clear residual fuel, ลด ITT และลดโอกาส fire หรือ hot restart

ข้อควรระวัง: ถ้า abort เพราะ hot/hung/no start โดยใช้ external power ไม่ควรดันทุรัง start ซ้ำด้วยอุปกรณ์เดิมโดยไม่วิเคราะห์สาเหตุ
จำเร็ว: Start ผิดปกติ → Stop fuel/start → Motor to clear

๔. Emergency Engine Shutdown on the Ground

EngineGround EmergencyFirewall

ความหมาย: การ shutdown เครื่องยนต์แบบฉุกเฉินบนพื้น ไม่ใช่ normal shutdown

ใช้เมื่อไหร่

  • Engine fire
  • Prop strike
  • Chip light
  • Aircraft กำลังจะออกนอก prepared surface
  • Ground emergency ที่ต้องหยุด engine ทันที

ลำดับคิด

  1. PCL - OFF หยุด command fuel/combustion
  2. Firewall Shutoff Handle - PULL isolate fuel, hydraulic fluid และ bleed air
  3. Emergency Ground Egress - As required ออกจากเครื่องถ้าอยู่ต่อไม่ปลอดภัย

ทำไม PCL OFF ก่อน firewall? PCL OFF คือสั่งเครื่องหยุด ส่วน firewall shutoff คือการตัดแหล่งของไหล/bleed ที่อาจเลี้ยงไฟหรือความเสียหายต่อ

จำเร็ว: OFF → PULL → OUT if required

๕. Takeoff Emergency Brief

BriefingTakeoffDecision

ความหมาย: การ brief ก่อน takeoff ว่า ถ้าเกิดเหตุระหว่าง takeoff roll หรือหลัง liftoff จะทำอะไร ใครทำอะไร และใช้เกณฑ์อะไรตัดสินใจ

Logic: ตอน takeoff เวลาคิดน้อยมาก ต้องตกลง “Stop / Go / Eject” ก่อนเริ่ม roll

ควรครอบคลุมอะไร

  • Abort criteria และ Max Abort Speed
  • ใครเป็น Pilot Flying / Pilot Monitoring
  • Loss of power ก่อน liftoff / หลัง liftoff
  • Runway remaining พอหรือไม่พอ
  • Emergency fuel / divert field / radio หรือ ICS failure
  • Ejection decision, command และ ISS mode

คิดเป็น 3 ช่วง

ช่วงแนวคิด
ก่อน Max Abort Speedถ้าควบคุมทิศทางไม่ได้หรือมี emergency ที่กระทบ safety ให้ abort
หลัง liftoff แต่ runway ยังพอเน้น land straight ahead และรักษา airspeed ที่ปลอดภัย
ไม่มี runway / aircraft ไม่ controllableคิด ejection early ไม่รอจน altitude/airspeed หมด
จำเร็ว: Brief ให้ชัดว่า Stop / Go / Eject ยังไง

๖. Premeditated Ejection Procedure

EjectionControlledSurvival

ความหมาย: การ ejection แบบมีเวลาคิดและเตรียม เช่น เครื่องยังควบคุมได้บางส่วน แต่รู้ว่า landing ไม่ปลอดภัยหรือไปต่อไม่ได้

Logic: แจ้ง crew → ตั้ง aircraft ให้เหมาะ → ลดอันตรายต่อคนบนพื้น → เตรียมร่างกาย → ดึง handle

ต่างจาก immediate ejection อย่างไร

ถ้า aircraft uncontrollable ต้อง eject ทันที เพราะการรออาจทำให้เสีย altitude, airspeed และ attitude จนโอกาสรอดลดลง แต่ถ้ายังมีเวลาให้ทำ controlled/premeditated ejection เพื่อเพิ่ม survival chance

ลำดับ controlled ejection ที่ควรเข้าใจ

  1. Notify crewmember ว่าจะ eject
  2. พยายามให้มี altitude และ airspeed ที่เหมาะสมตาม manual/SOP
  3. Transmit distress call และ set transponder ตาม procedure
  4. Stow loose equipment
  5. Visor down, mask/helmet/chin strap tight
  6. Check leg restraints และ harness locked
  7. Turn aircraft ไปพื้นที่ไม่ใช่ชุมชนถ้าทำได้
  8. PCL OFF
  9. จัดท่า: head back, back/shoulders against seat, elbows in, legs flat
  10. Pull ejection handle
จุดสำคัญ: อย่ารอจน sink rate สูงหรือ altitude ต่ำเกินไป เพราะ seat performance และ survival margin จะลดลงมาก
จำเร็ว: Tell → Turn → Tighten → Throttle OFF → Position → Pull

๗. Hand Over / Take Over Procedure

Crew CoordinationFlight ControlsICS

ความหมาย: การส่งมอบ/รับมอบ control ระหว่างนักบินสองคนให้ชัดเจน เพื่อป้องกัน “ไม่มีใครบิน” หรือ “สองคนแย่งกันบิน”

Logic: ต้องมี positive transfer of control ทั้งคำพูดและการกระทำ

รูปแบบ verbal 3-way

  1. คนส่ง: “You have control.”
  2. คนรับ: “I have control.”
  3. คนส่งยืนยัน: “You have control.”

ถ้า ICS ใช้ไม่ได้

  • คนที่ต้องการ control: shake control stick
  • คนที่ relinquish control: raise both hands

การ coordinate ไม่ใช่แค่ flight controls แต่รวมถึง canopy, brakes, radios, avionics และ subsystem actions ด้วย

จำเร็ว: Say it → Confirm it → Show it

๘. Use of Firewall Shutoff Handle

FuelHydraulicBleed AirEngine Fire

ความหมาย: Handle ที่ใช้ isolate engine compartment โดยตัดของไหลและ bleed air ที่อาจเลี้ยงไฟหรือความเสียหาย

เมื่อดึงแล้วตัดอะไร

  • Fuel ไป engine
  • Hydraulic fluid ไป engine area
  • Bleed air จาก engine

ใช้เมื่อไหร่

  • Engine fire
  • Emergency engine shutdown
  • Prop strike หรือ chip light
  • Ground emergency ที่ต้อง isolate engine
Logic: ถ้า fire/damage อยู่หน้า firewall แล้ว fuel/hydraulic/bleed ยัง feed อยู่ ปัญหาอาจลามหรือรุนแรงขึ้น Firewall shutoff คือการตัดเสบียง
จำเร็ว: Firewall handle = isolate engine from fuel / hydraulic / bleed

๙. Intercom Failure

ICSAudioCrew Coordination

ความหมาย: ระบบสื่อสารระหว่าง cockpit หรือ audio system ใช้ไม่ได้ ทำให้ crew coordination เสี่ยงทันที โดยเฉพาะการส่งมอบ control และ emergency coordination

ICS ทำหน้าที่อะไร

  • คุยระหว่าง front/rear cockpit
  • ฟัง radio และ nav audio
  • คุยกับ ground crew
  • ปรับ audio/volume ผ่าน audio control panel

ลำดับแก้ไขที่ควรเข้าใจ

  1. Switch COMM lead to auxiliary cord ใช้สายสำรองใน cockpit ที่ affected
  2. Check AUDIO circuit breaker ถ้า open ให้ reset ตามข้อกำหนด
  3. EMR/NRM switch - EMR ถ้ายังไม่ได้ audio ให้ใช้ emergency/raw audio path ตาม procedure
  4. ใช้ non-verbal transfer ถ้าต้อง hand over/take over control
ข้อจำกัดของ EMR: เป็น alternate/raw audio path และอาจทำให้ audio บางช่องไม่ได้ยินหรือไม่ได้ amplify ต้องเข้าใจข้อจำกัดก่อนใช้
จำเร็ว: Cord → CB → EMR → Non-verbal control transfer
