One-line Summary

PMU Failure คือสถานการณ์ที่ Power Management Unit ใช้งานไม่ได้หรือถูกตัดออกจาก automatic control ทำให้ระบบ engine/propeller บางส่วนเข้าสู่ manual mode นักบินต้องควบคุม power และ monitor engine limits เองมากขึ้น โดยเฉพาะ torque, ITT และ N1

Aircraft อาจยังบินได้และ engine ไม่ได้ดับทันที แต่ protection/automatic limiting สำคัญหายไป จึงต้องลด power เท่าที่ทำได้อย่างปลอดภัย, isolate PMU ที่ไม่น่าเชื่อถือ, reset ตามเงื่อนไขของ manual และถ้า reset ไม่สำเร็จให้ยุติภารกิจและลงจอดเมื่อเหมาะสม

Indications หลัก

  • PMU FAIL Power Management Unit failure
  • PMU STATUS PMU detected/accommodated a fault หรือ WOW switch failure
  • อาจมี step change in engine power
  • สูญเสีย automatic control/limiting ของ torque, ITT, N1

เมื่อ PMU OFF mechanical overspeed governor อาจ allow NP range ประมาณ 98% ถึง 102%

Learning Objectives

เมื่อเรียนจบหน้านี้ ผู้เรียนควรสามารถ:

  • อธิบายหน้าที่ของ PMU ใน normal operation ได้
  • อธิบายความต่างระหว่าง PMU Failure และ PMU Fault ได้
  • ระบุระบบที่เกี่ยวข้อง ได้แก่ PMU, FMU, PCL, PIU, propeller governor, PMA/battery bus และ WOW switches ได้
  • บอก indications บน EICAS/CAS ที่เกี่ยวข้องได้
  • อธิบายว่าทำไมต้องตั้ง PCL ไปที่ minimum practical ก่อน switch/reset PMU ได้
  • อธิบายว่าทำไม reset PMU ไม่ควรทำเมื่อมี uncommanded power changes ที่ไม่ใช่ anticipated step change ได้
  • อธิบายผลต่อ landing rollout/taxi จากการไม่มี ground idle ได้
  • ใช้ memory aid เพื่อจำ logic ของ checklist ได้

PMU Normal Operation: ปกติ PMU ทำอะไร

Power Management Unit (PMU) เป็น controller สำคัญที่ช่วยจัดการ engine power และ propeller control ให้สัมพันธ์กับตำแหน่งของ PCL และสภาพการบิน โดยมีหน้าที่หลัก 4 กลุ่มใหญ่

1. Schedule power ตาม PCL

Manual อธิบายว่า power setting มีลักษณะเป็น linear กับตำแหน่ง PCL ภายใน operating envelope เช่น ถ้า max power available ณ สภาพนั้นคือ 1000 SHP ที่ max PCL และ PCL อยู่ครึ่งทางระหว่าง idle กับ max ระบบจะ schedule ประมาณ 500 SHP

ความหมาย: นักบินควบคุม power ด้วย PCL แต่ PMU ช่วยแปลคำสั่ง PCL ให้เป็น fuel/propeller behavior ที่เหมาะสม

2. ควบคุม acceleration / deceleration

PMU ช่วยควบคุมการเร่งและลดรอบของเครื่องยนต์ ทำให้สามารถขยับ PCL ได้รวดเร็วตามต้องการใน altitude ต่าง ๆ โดยลด propeller effects และหลีกเลี่ยง thrust surge ขนาดใหญ่ขณะ engine acceleration

ความหมาย: เมื่อ PMU ปกติ นักบินมี workload ต่ำกว่า เพราะ PMU ช่วยทำให้ response smooth และช่วยป้องกัน limit

3. ควบคุม propeller ผ่าน PIU

PMU ร่วมกับ Propeller Interface Unit (PIU) ควบคุม blade angle/pitch และ propeller speed NP โดยใช้ oil pressure ไปยัง propeller pitch change piston เพื่อ maintain propeller speed ประมาณ 2000 RPM หรือ 100% NP ใน flight condition ส่วนใหญ่

ความหมาย: PMU ไม่ได้คุมแค่ fuel แต่ยังเกี่ยวกับ propeller governing และการลดผลกระทบจาก propeller ต่อ aircraft handling

4. ช่วย start sequencing และ protection บางส่วน

ระหว่าง ground auto start, PMU monitor N1, ITT และ fuel flow และสามารถ abort start เพื่อหลีกเลี่ยง no start, hung start หรือ hot start ตามเงื่อนไขที่ manual กำหนด

ข้อจำกัด: ระหว่าง airstart PMU ให้ sequencing ได้ แต่ manual ระบุว่า automatic N1, ITT และ torque limit protection ไม่ available นักบินต้อง monitor parameter เอง

PMU Power Source

PMU ใช้ไฟจาก Permanent Magnet Alternator (PMA) ที่ติดตั้งบน reduction gearbox โดย PMA จ่าย 32 VAC แล้ว PMU แปลงเป็น DC หาก propeller RPM ต่ำกว่า 40–50% NP หรือ PMA fail ระบบจะ switch ไปใช้ 28 VDC battery bus อัตโนมัติ

ระบบที่เกี่ยวข้องกับ PMU Failure

ระบบ / Component ทำหน้าที่อะไรในภาวะปกติ เกี่ยวข้องกับ PMU Failure อย่างไร ผลที่นักบินต้องสนใจ
PMU
Power Management Unit		 ควบคุม power schedule, acceleration/deceleration, engine limit logic และ propeller command ร่วมกับ PIU เมื่อ fault รุนแรงจนไม่สามารถ set requested power หรือ respect engine limits ได้ ระบบ revert เป็น manual mode หรือ pilot อาจ switch OFF เอง automatic limiting ของ torque/ITT/N1 ไม่ available ต้อง monitor engine parameters เอง
PCL
Power Control Lever		 เป็น primary pilot input สำหรับ engine power เชื่อมกับ FMU ผ่าน cable/rod system ใน manual mode PCL schedules fuel directly to engine through FMU ต้องขยับอย่างระมัดระวัง หลีกเลี่ยง overshoot ของ torque, ITT, N1 และระวัง power shift
FMU
Fuel Management Unit		 จ่าย fuel ที่ต้องการไป flow divider และคืน excess fuel ไป pump เมื่อ PMU manual mode, fuel management revert ไป nominal/manual control และ PCL มีผลโดยตรงมากขึ้น engine ยังตอบสนองได้ แต่ไม่มี PMU protection แบบเดิม
PIU
Propeller Interface Unit		 เพิ่ม/จัดการ oil pressure ไป propeller pitch change mechanism ตามคำสั่ง PMU PMU failure ทำให้ propeller governing ไม่อยู่ภายใต้ electronic control ปกติ propeller protection ไปพึ่ง mechanical overspeed governor
Mechanical Overspeed Governor เป็น mechanical backup ที่ modulate oil pressure เพื่อจำกัด NP เมื่อ PMU lost/deactivated governor นี้รักษา NP ที่ประมาณ 100±2% อาจเห็น NP range 98–102% และอาจมี torque surge เมื่อ propeller accelerate จาก low power
PMA / Battery Bus PMA power PMU; battery bus เป็น backup เมื่อ prop RPM ต่ำหรือ PMA fail failure อาจเกี่ยวกับ PMU power/control path จึงมี checklist ให้ check/reset IGN, START, PMU circuit breakers ต้องเข้าใจว่า PMU เป็นระบบ electrical/control system ไม่ใช่ mechanical fuel control อย่างเดียว
Weight-on-Wheels Switches ให้ PMU แยก flight mode กับ ground mode PMU STATUS in flight เกี่ยวข้องกับ discrepancy/mismatch ของ WOW switches ได้ ถ้า PMU ไม่ revert เป็น ground mode หลัง landing, ground idle จะไม่ available และ landing distance เพิ่มขึ้น
EICAS / CAS แสดง warning/caution/advisory แสดง PMU FAIL และ PMU STATUS เพื่อบอก failure/fault state ต้องแยก PMU Failure กับ PMU Fault จาก combination ของ annunciations

PMU Failure / PMU Inoperative เกิดอะไรขึ้น

Manual ระบุว่า PMU self-monitoring อยู่ตลอดเวลาและสามารถ identify/accommodate fault ได้หลายแบบ แต่ถ้า fault ทำให้ PMU ไม่สามารถ set requested power หรือ respect engine limits ได้ หรือ pilot switch PMU OFF ระบบจะ revert ไป manual mode

KEY FAILURE EFFECT

เมื่อเข้าสู่ PMU Failure / manual mode จะสูญเสีย automatic control ของ torque, ITT และ N1 นักบินต้องควบคุมและ monitor parameters เหล่านี้เอง

Indication ที่บอก manual mode

Manual ระบุว่า illumination พร้อมกันของ PMU FAIL warning และ PMU STATUS caution บน EICAS แสดงว่าระบบอยู่ใน manual mode

Power step change

อาจเกิด step change in engine power เมื่อ FMU revert ไป nominal/manual setting โดย transition โดยทั่วไป smooth และ controllable. ปกติ step change มักไป lower power แต่ใน low altitude/cold condition อาจเพิ่ม power ได้

ตัวเลข power step change

Manual ให้ maximum increase ประมาณ 280 shp และ maximum decrease ประมาณ 550 shp

NP ไม่ควร step change

Manual ระบุว่า no step change in NP occurs เพราะ overspeed governor ถูก reset ไปที่ 100%

Engine / Propeller Behavior หลัง PMU OFF หรือ PMU Failure

  • Fuel side: PCL schedules fuel directly to engine through FMU
  • Limit side: pilot ต้องระวังไม่ให้ N1, temperature/ITT และ torque เกิน limits
  • Response side: engine acceleration/deceleration characteristics โดยพื้นฐานยังใกล้เคียงเดิม แต่ต้องใช้ความระมัดระวังมากขึ้น
  • Propeller side: propeller operating on mechanical overspeed governor
  • Surge risk: manual ระบุว่า torque surge can be expected anytime the propeller is accelerated to governing speed from low power setting เช่น takeoff roll หรือ go-around

PMU Failure Checklist Logic

ใช้ procedure นี้เมื่อ PMU becomes inoperative and reset is desired หรือเมื่อ PMU FAIL warning illuminates โดย indications อาจรวมถึง PMU FAIL + PMU STATUS และ possible step change in engine power

BEFORE RESET / SWITCHING PMU OFF

Manual เน้นให้ set power ที่ lowest practical setting ก่อน resetting PMU หรือ switching PMU OFF เพื่อ minimize power shift

PCL — Minimum practical for flight

ลด PCL ไปที่ power ต่ำที่สุดที่ยังคงปลอดภัยต่อการบิน ควบคุม aircraft ได้ และไม่ทำให้สถานการณ์แย่ลง

ทำเพื่ออะไร:
  • ลดขนาดของ power shift ถ้า PMU ถูก switch OFF หรือ reset แล้ว fuel schedule เปลี่ยน
  • ลดโอกาสเกิด torque/ITT/N1 overshoot เพราะ automatic limiting หายไป
  • ลด thrust surprise และช่วยให้ aircraft controllability ดีขึ้น
  • สร้างสภาพที่ปลอดภัยกว่าในการ isolate/reset controller
PMU switch — OFF

ตัด PMU ออกจาก automatic control และยอมรับ manual mode เพื่อหยุด controller ที่ไม่น่าเชื่อถือไม่ให้มีผลต่อ engine/propeller command

ทำเพื่ออะไร:
  • เอา PMU ที่ malfunction หรือไม่สามารถ respect limits ได้ออกจาก control loop
  • ให้ fuel management อยู่ใน manual/nominal setting ที่ predictable มากกว่า PMU ที่ fault
  • ทำให้ pilot คุม power ผ่าน PCL/FMU โดยตรงมากขึ้น
  • propeller governing ถูก provide โดย mechanical overspeed governor
DO NOT RESET PMU IN THIS CASE

ถ้า PMU failure มาพร้อม uncommanded power changes ที่ไม่ใช่ anticipated step change จากการ revert เป็น manual mode, manual ระบุว่า do not reset PMU และให้ refer ไป procedure ที่เกี่ยวข้องกับ Uncommanded Power Changes / Loss of Power / Uncommanded Propeller Feather

NOTE

Pilot ควรพิจารณาขยับ PCL ผ่าน full range of motion เพื่อ determine power available. ความหมายเชิงการบินคือ ต้องรู้ว่า power ที่เหลือพอสำหรับรักษาระดับ, climb, approach, waveoff/go-around หรือ profile ที่ต้องบินต่อไปหรือไม่

To Reset PMU

IGN, START, and PMU circuit breakers — Check and reset if necessary

ตรวจ circuit breakers ที่เกี่ยวข้องกับ ignition, start และ PMU บริเวณ left front console และ reset หากจำเป็นตามแนวทางของ manual/SOP

ทำเพื่ออะไร:
  • ตรวจว่าปัญหาอาจเกิดจาก electrical/control circuit ที่ตัดออกหรือไม่
  • restore power/control path ที่จำเป็นต่อ PMU/starting/ignition logic
  • ถ้า CB trip เป็นสาเหตุ การ reset อาจทำให้ PMU กลับมา online ได้

ข้อควรระวัง: การ reset CB ต้องทำตามนโยบายหน่วยและ current checklist เท่านั้น เพราะการ reset faulted circuit โดยไม่เข้าใจเหตุผลอาจเพิ่มความเสี่ยงทางไฟฟ้า/ระบบ

PMU switch — NORM

นำ PMU กลับเข้าตำแหน่ง normal เพื่อพยายาม reset. Manual ระบุว่า attempt second reset if necessary

ทำเพื่ออะไร:
  • ให้ PMU reboot/reinitialize และกลับมาทำ automatic control ถ้า fault clear แล้ว
  • ถ้าสำเร็จ PMU FAIL/PMU STATUS ควร clear ตามสถานการณ์
  • ถ้าไม่ clear ต้องถือว่า automatic torque/ITT/N1 limiting ยังไม่ available
IF ANNUNCIATORS DO NOT CLEAR

ถ้าการกระทำข้างต้นไม่ clear annunciator(s), pilot ต้องตระหนักว่า automatic torque, ITT และ N1 limiting ไม่ available

If PMU Reset Is Unsuccessful

PMU switch — OFF

กลับไป manual mode และไม่ใช้ PMU ที่ยังมี fault เป็น controller

ทำเพื่ออะไร:
  • ป้องกัน PMU ที่ fault ส่ง command ที่ไม่น่าเชื่อถือ
  • ทำให้สภาพระบบ predictable ภายใต้ manual mode มากขึ้น
  • บังคับให้ pilot mindset เปลี่ยนไป monitor engine limits เองตลอดเวลา
Land as soon as practical

ยุติภารกิจและลงจอดเมื่อสถานการณ์เหมาะสม โดยไม่ลาก mission ต่อโดยไม่จำเป็น

ทำเพื่ออะไร:
  • ลด exposure time กับ engine/propeller system ที่ไม่มี PMU automatic limiting
  • ลดโอกาสต้องทำ power changes จำนวนมากภายใต้ manual mode
  • ให้ crew เตรียม landing ด้วย idle N1 ที่สูงขึ้นและ braking/rollout distance ที่เปลี่ยนไป

RESET OK ถ้า annunciations clear

PMU อาจกลับมา online และ automatic control/limiting อาจกลับมาทำงานตามปกติ แต่ยังควร monitor parameters อย่างใกล้ชิดและปฏิบัติตาม SOP/maintenance guidance หลัง flight

RESET FAIL ถ้า annunciations ไม่ clear

ให้ PMU OFF, ถือว่า torque/ITT/N1 automatic limiting ไม่ available, plan landing as soon as practical และคำนึงถึง increased landing distance เพราะ ground idle ไม่ available

PMU Failure vs PMU Fault: ต้องแยกให้ชัด

สถานะ Annunciation ความหมาย Action / Implication
PMU Failure / Manual Mode PMU FAIL + PMU STATUS PMU inoperative หรือถูกตัดออกจาก automatic mode. Automatic control/limiting ของ torque, ITT, N1 lost ทำ PMU Failure procedure. ถ้า reset ไม่สำเร็จ PMU OFF และ land as soon as practical
PMU Fault PMU STATUS โดย PMU FAIL extinguished PMU detected/accommodated a fault แต่ PMU remains online และ continues to function ขึ้นกับ ground/inflight condition. ต้องพิจารณา WOW switch discrepancy และ ground idle issue

PMU Fault — On Ground

  • ถ้า PMU STATUS caution illuminates after landing: notify maintenance
  • Action: PMU switch — OFF, then NORM
  • ถ้า PMU STATUS caution ยัง illuminated: confirm source of fault prior to flight

PMU Fault — Inflight

Manual ระบุว่า PMU detected a discrepancy in the weight-on-wheels switch และ a reset is not possible

LANDING IMPLICATION

Ground idle จะไม่ available ระหว่าง landing rollout และ taxi. ต้อง plan increased landing distances เนื่องจาก idle N1 สูงขึ้นประมาณ 67%

NOTE

เมื่อ landing gear ถูก extend แล้ว weight-on-wheels circuit malfunction อาจป้องกันไม่ให้ gear retract ได้

Landing / Rollout / Taxi Impact

ผลกระทบสำคัญหลัง PMU reset ไม่สำเร็จหรือกรณี PMU STATUS ที่เกี่ยวข้องกับ WOW switch คือ ground idle may not be available

ทำไม idle สูงขึ้นจึงสำคัญ

ปกติ PMU ใช้ WOW switches เพื่อเปลี่ยนระหว่าง flight mode กับ ground mode. Ground mode idle ประมาณ 60–61% N1 ส่วน flight mode idle ขั้นต่ำประมาณ 67% N1. ถ้า PMU ไม่ revert เป็น ground mode หลัง touchdown, engine idle thrust จะสูงกว่าปกติ

ผลต่อ landing distance

idle N1 สูงขึ้นทำให้ residual thrust สูงขึ้น ส่งผลให้ aircraft มีแนวโน้ม float/roll ยาวขึ้นหรือ braking demand สูงขึ้น ต้อง plan increased landing distance และใช้ runway/landing technique ตาม manual/SOP

DO NOT UNDERESTIMATE

แม้ engine ยังทำงานได้ แต่การไม่มี ground idle เปลี่ยน energy state หลัง touchdown อย่างมีนัยสำคัญ โดยเฉพาะ runway สั้น, contaminated runway, tailwind, high gross weight หรือ landing ที่ต้องหยุดระยะจำกัด

Go-around / Power Change Consideration

Manual ระบุว่าเมื่อ PMU OFF และ propeller อยู่บน overspeed governor อาจเกิด torque surge ได้เมื่อ propeller accelerate ไป governing speed จาก low power setting เช่น takeoff roll หรือ go-around ดังนั้นการขยับ PCL ต้อง smooth, deliberate และ monitor parameters อย่างใกล้ชิด

Pilot Mental Model / Memory Aid

ลดก่อน → ตัดก่อน → อย่า reset ถ้า power เปลี่ยนเองผิดปกติ → Check CB → ลอง NORM → ไม่หาย OFF → Land practical

1. ลดก่อน

ตั้ง PCL minimum practical เพื่อลด power shift และลดโอกาสเกิน limit

2. ตัดก่อน

PMU OFF เพื่อเอา controller ที่ไม่น่าไว้ใจออกจาก loop

3. ค่อย reset

check CB แล้ว PMU NORM เพื่อดูว่าระบบกลับมาได้หรือไม่

4. ไม่หาย = manual mindset

PMU OFF และ monitor torque/ITT/N1 เอง

5. ลงเมื่อ practical

ลด exposure และเตรียม landing distance เพิ่มขึ้น

6. แยก fault/failure

PMU STATUS อย่างเดียวไม่เท่ากับ PMU FAIL + STATUS

Quick Check / Oral Exam Style

1. PMU Failure indication หลักคืออะไร?
PMU FAIL warning illuminates และโดยปกติ simultaneous PMU STATUS caution บน EICAS บ่งชี้ว่า system อยู่ใน manual mode. อาจมี step change in engine power ด้วย
2. ทำไมขั้นแรกต้อง PCL — Minimum practical for flight?
เพื่อลด power shift ระหว่าง PMU OFF/reset, ลดโอกาส torque/ITT/N1 overshoot และช่วยให้ aircraft controllability ดีขึ้นก่อน isolate/reset controller
3. เมื่อ PMU อยู่ manual mode ใคร schedule fuel?
PCL schedules fuel directly to the engine through the FMU. PMU automatic limiting ของ torque/ITT/N1 ไม่ available นักบินต้อง monitor เอง
4. Propeller governing หลัง PMU failure พึ่งอะไร?
พึ่ง mechanical overspeed governor ซึ่งจะ provide propeller governing/overspeed protection โดย NP อาจอยู่ประมาณ 100±2% หรือ 98–102% เมื่อ PMU OFF
5. ถ้ามี uncommanded power changes ที่ไม่ใช่ anticipated step change ควร reset PMU หรือไม่?
ไม่ควร reset PMU. Manual ให้ refer procedure ที่เกี่ยวข้องกับ Uncommanded Power Changes / Loss of Power / Uncommanded Propeller Feather
6. PMU STATUS อย่างเดียวใน flight สำคัญอย่างไร?
บ่งชี้ discrepancy/mismatch ใน weight-on-wheels switch และ reset is not possible. ต้อง plan ว่า ground idle จะไม่ available หลัง landing และ landing distance อาจเพิ่มขึ้นเพราะ idle N1 ประมาณ 67%
7. ถ้า PMU reset ไม่สำเร็จ checklist ไปทางไหน?
PMU switch OFF แล้ว land as soon as practical พร้อม awareness ว่า automatic torque/ITT/N1 limiting ไม่ available และ ground idle จะไม่ available ระหว่าง landing rollout/taxi
